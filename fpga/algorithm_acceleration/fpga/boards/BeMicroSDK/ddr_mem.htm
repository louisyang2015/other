Memory Data

Part Number: MT46H32M16LFBF-5

Annotated Datasheet

Parameter Decisions:

The preset used is: MT46V16M16TG-5B. So the default values come from that preset profile.

Output clock pairs from FPGA: 1 - there's only one memory device

Memory chip selects: 1 - only 1 pin on the schematic

Memory DQ width: 16

Memory burst length: 4 - Datasheet page 1 says burst length is programmable: 2, 4, 8, or 16. The "full rate" controller is used, which supports only burst length of 4.

Memory burst order: Interleaved - Datasheet page 52 says the burst order can be programmed to be interleaved.

Memory CAS latency: 2 - Datasheet page 53 says the CAS latency can be 2 or 3.

Column Address Width: 10 - Datasheet page 2

Row Address Width: 13 - Datasheet page 2

Bank Address Width: 2 - Datasheet page 2

Chip selects per DIMM: 1 - Only 1 memory chip

DQ bits per DQS bit: 8 - Datasheet page 13

Precharge Address bit: 10 - Datasheet page 83

Drive DM pins from FPGA: Yes - because schematic shows that DM pins go to FPGA

Timing Information:

t_INIT: 200us - Datasheet page 48.

t_MRD: 16 ns - Datasheet page 27; 2 * 8ns

t_RAS: 45 ns - Datasheet page 27

t_RCD: 15 ns - Datasheet page 27

t_RP: 15 ns - Datasheet page 28

t_REFI: 7.8 us - Datasheet page 27; There are two t_REFI values. This particular memory is 512 Mbit

t_RFC: 72 ns - Datasheet page 27

t_WR: 15 ns - Datasheet page 28

t_WTR: 2 clock cycles - Datasheet page 28

t_AC: 700 ps - Datasheet on page 26 says 2 ~ 5 ns, but Altera manual says only 300 ~750 ps is allowed. The MegaWizard will only accept 300 ~ 1000 ps. The two definitions must have differed by half a clock cycle (4ns).  So I just use the default value of 700 ps.

t_DH: 530 ps - Datasheet page 26 list 0.48 ~ 0.58 ns, depending on the slew rate. So a midpoint is picked.

t_DQSCK: 450 ps - same issue as t_AC: the datasheet on page 26 says 2 ~ 5 ns, but the manual gives a range of 100~750 ps. So the two are likely to be off by half a clock cycle. Default value is used.

t_DQSQ: 400 ps - Datasheet page 26

t_DQSS: 0.25 clock cycle - Datasheet page 26 says 0.75 clock cycle, but the manual gives a range of 0 ~ 0.3 clock cycle. So it's one of those diff by half a clock cycle things.

t_DS: 530 ps - Datasheet page 26 provides two values, for two different slew rates.

t_DSH: 0.2 clock cycle - Datasheet page 26

t_DSS: 0.2 clock cycle - Datasheet page 26

t_IH: 1.1 ns - Datasheet page 27 lists two hold times for two slew rates. The longer of the two hold times is used.

t_IS: 1.1 ns - Datasheet page 27 lists two set up times and the longer of the two is used.

t_QHS: 0.5 ns - Datasheet page 27

t_RRD: 10 ns - Datasheet page 28

Maximum memory frequency for CAS latency: 83.333 MHz for CAS latency of 2 and 200 MHz for CAS latency of 3 - Datasheet page 26
