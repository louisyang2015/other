OnChip Memory

For read-only memory, an initialization file is needed. When using the "hex" format, it seems the Quartus software will warn of "overflow" unless the "hex" file is 1 word per line. This behavior is observed during the FFT and Array Processor projects, where the twiddle factor needs to be stored as read-only memory.

 

FIFO

In the Array Processor project, a dual clock, look ahead FIFO is used. In that project, after doing a read or write, it seems that you have to wait a cycle for the rdempty / wrfull flag to be true. In other words, it's not possible to read / write continuously, because the "rdempty" and "wrfull" flags are not immediately correct after each read/write.
