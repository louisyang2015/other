To accommodate the FFT, the x0_sel0 output is always delayed two clock cycles relative to the rdaddress input. In the datapath, the x0_sel0 output is always required to launch two cycles after the launch of the rdaddress.

For example, during FFT stage 0, the two cycle delay is created as:

  • x0_sel0_delay <= x0_sel0_ctrl;
  • x0_sel0 <= x0_sel0_delay;
FFT Stage x0_sel0_delay
0 x0_sel0_delay <= x0_sel0_ctrl;
x1_sel0_delay <= x1_sel0_ctrl;
1 x0_sel0_delay <= rdaddress[0];
x1_sel0_delay <= ~rdaddress[0];
2 x0_sel0_delay <= rdaddress[1];
x1_sel0_delay <= ~rdaddress[1];
3 x0_sel0_delay <= rdaddress[2];
x1_sel0_delay <= ~rdaddress[2];
4 x0_sel0_delay <= rdaddress[3];
x1_sel0_delay <= ~rdaddress[3];
5 x0_sel0_delay <= rdaddress[4];
x1_sel0_delay <= ~rdaddress[4];
6 x0_sel0_delay <= rdaddress[5];
x1_sel0_delay <= ~rdaddress[5];
7 x0_sel0_delay <= rdaddress[6];
x1_sel0_delay <= ~rdaddress[6];
8 x0_sel0_delay <= rdaddress[7];
x1_sel0_delay <= ~rdaddress[7];
9 x0_sel0_delay <= rdaddress[8];
x1_sel0_delay <= ~rdaddress[8];
default: x0_sel0_delay <= x0_sel0_ctrl;
x1_sel0_delay <= x1_sel0_ctrl;
