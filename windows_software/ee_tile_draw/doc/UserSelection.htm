UserSelection

User's current selections are in this class.

From the user's perspective:

The UserSelection class records the user selection so that the following design objectives are met:

  • Say the user has been drawing the wires with a foreground color C1 and background color C2. Then the user switch to drawing something else, it's still the same foreground color C1 and background color C2.
  • Say the user has been using vertical wires (orientation_id = 0). The user now draws a resistor. Then the user switches back to the wires after drawing the resistor --- the wires being used are still of the vertical kind.

The first requirement is easy --- more or less a global "Color" variable

The second requirement means storing a "type_id" and an "orientation_id" for each "class_id"

The UserSelection class has an array of of pointers to "ClassDefault" objects. Each ClassDefault object is a triplet of (class_id, type_id, orientation_id). The user can add to this array and query this array for the type_id, and orientation_id that is associated with a certain class_id.

The UserSelection class is very similar to the ImageList class in that both classes uses "shell_sort( )" to sort out an array of pointers and a binary search to search that same array of pointers.
