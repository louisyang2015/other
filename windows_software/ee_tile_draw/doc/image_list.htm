ImageList

This class will load the various ".wmf" files first, and load it once. After start up, the image list will return a pointer to an image when the image is requested by ID. A separate method, "get_rotate", will get the image for the rotation button.

The ImageList itself is quite easy to understand, no need to explain it much. ImageList also contains GDI+ startup code in the constructor and the shutdown code in the destructor. The reason is that the memory pointers to Image objects needs to be de-allocated first before the GDI+ shutdown. Once the GDI+ shutdown code has executed, I could not use "delete image" to de-allocate the memory.

While writing the ImageList class, I have decided to use "Shell sort" as my basic sorting algorithm. So this ImageList class has a well documented shell sort for me to save for the future.
