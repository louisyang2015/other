Rectangles and Regions

RECT functions

FillRect (hdc, &rect, hBrush);
FrameRect (hdc, &rect, hBrush); //draw border only
InvertRect (hdc, &rect);

SetRect (&rect, xLeft, yTop, xRight, yBottom);

To obtain the intersection of two rectangles:

IntersectRect (&DestRect, &SrcRect1, &SrcRect2);

To obtain the union of two rectangles:

UnionRect (&DestRect, &SrcRect1, &SrcRect2);

To determine whether a point is in a rectangle:

bInRect = PtInRect (&rect, point);

 

Creating Regions

HRGN hRgn;
hRgn = CreateRectRgn (xLeft, yTop, xRight, yBottom);
hRgn = CreateRectRgnIndirect (&rect);

hRgn = CreateEllipticRgn (xLeft, yTop, xRight, yBottom);
hRgn = CreateEllipticRgnIndirect (&rect);

hRgn = CreatePolygonRgn (&point, iCount, iPolyFillMode);

point = an array of structure of type POINT
iPolyFillMode = ALTERNATE or WINDING

 

Deleting Regions

DeleteObject (hRgn);

 

Combining Regions

iRgnType = CombineRgn (nDestRgn, hSrcRgn1, hSrcRgn2, iCombine);

iCombine can be:
RGN_AND
RGN_OR
RGN_XOR
RGN_DIFF
RGN_COPY

iRgnType can be:
NULLREGION
SIMPLEREGION
COMPLEXREGION
ERROR

 

Drawing a region

FillRgn (hdc, hRgn, hBrush);
FrameRgn (hdc, hRgn, hBrush, xFrame, yFrame);
InvertRgn (hdc, hRgn);
PaintRgn (hdc, hRgn);

all of the above use logical coordinates

 

Using Regions to do clipping

SelectObject (hdc, hRgn);
SelectClipRgn (hdc, hRgn);

The clipping region is in device coordinates. Also, the GDI makes a copy of the clipping region, so you can delete the region object after you select it in the device context.

 

Using Regions with WM_PAINT

to erase the client area:

InvalidateRect (hwnd, NULL, TRUE);

or

InvalidateRgn (hwnd, hRgn, bErase);

To validate a region:

ValidateRgn (hwnd, hRgn);
