WM_CHAR

The keyboards generate WM_KEYDOWN, WM_KEYUP messages --- if the key the user typed in is a printable character, Windows will add in the WM_CHAR message. Since this is something that Windows add in to compensate for locale differences, not all keys will cause windows to add in a WM_CHAR message.

The character code delivered with the WM_CHAR message is:

(TCHAR) wParam

In addition to the basic alphabet and numbers, WM_CHAR is also generated for:

'\b' --- Backspace
'\t' --- Tab
'\n' --- ctrl-Enter
'\r' --- Enter

 

WM_CHAR sample code:

case WM_CHAR:
    switch (wParam)
    {
    case `\b':          // backspace

        break ;
    case `\t':          // tab

        break ;

    case `\n':          // linefeed
   
        break ;

    case `\r':          // carriage return

        break ;

    default:            // character codes

        break ;
    }
    return 0 ;

 

Other notes:

In addition to WM_CHAR, there's also WM_DEADCHAR, WM_SYSCHAR, and WM_SYSDEADCHAR.
