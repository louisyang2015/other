The Edit Control

Click here for a program that demonstrates using an edit control

the window class name: "edit"

window styles:

ES_LEFT --- left justified
ES_RIGHT --- right justified
ES_CENTER --- centered
ES_MULTILINE --- multi-line control
ES_AUTOHSCROLL
ES_AUTOVSCROLL
WS_HSCROLL --- "AUTOHSCROLL" is the ability, this is the actual scroll bar
WS_VSCROLL
WS_BORDER --- by default, edit controls don't have a border
ES_NOHIDESEL --- to have selection highlighted even when the edit control does not have the input focus

 

messages passed to the parent:

they are WM_COMMAND messages, in the format:

LOWORD (wParam) --- Child window ID
HIWORD (wParam) --- Notification code
lParam --- Child window handle

useful notificaiton codes:

EN_UPDATE --- Edit control's contents have changed

EN_ERRSPACE, EN_MAXTEXT --- edit control out of space error

 

Manipulating the control:

changing the control's text: SetWindowText, GetWindowText

SendMessage (hwndEdit, WM_CUT, 0, 0);
SendMessage (hwndEdit, WM_COPY, 0, 0);
SendMessage (hwndEdit, WM_CLEAR, 0, 0);
SendMessage (hwndEdit, WM_PASTE, 0, 0);

SendMessage (hwndEdit, EM_GETSEL, (WPARAM) &iStart, (LPARAM) &iEnd);
Note: the ending position of a selection is actually the position of the last character plus 1.

SendMessage (hwndEdit, EM_SETSEL, iStart, iEnd);

SendMessage (hwndEdit, EM_REPLACESEL, 0, (LPARAM)szString);

how many lines are there?
iCount = SendMessage (hwndEdit, EM_GETLINECOUNT, 0, 0);

get an index into a given line; use iLine = -1 for the current line
iOffset = SendMessage (hwndEdit, EM_LINEINDEX, iLine, 0);

get the length of a given line
iLength = SendMessage (hwndEdit, EM_LINELENGTH, iLine, 0);

get the text of a given line
iLength = SendMessage (hwndEdit, EM_GETLINE, iLine, (LPARAM) szBuffer);
