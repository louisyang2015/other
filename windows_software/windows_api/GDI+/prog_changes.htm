Changes in the Programming Model

A lot less Device Context appearance --- Instead, you create a Graphics object and then invoke its methods.

No more "SelectObject" calls --- The Graphics object is not attached to a particular pen, brush, path, image, or font. So you need to supply these things each time you invoke a "Graphics" method.

No more current position

Separate methods for draw and fill --- previously, GDI had methods that does both draw and fill

The Region object's constructor only accepts rectangles --- The older GDI used different functions to generate non-rectangular regions. Now, to generate a region that is not a rectangle, you should construct a GraphicsPath object. The path object is not destroyed in the construction of a region. The "Region" class has "Union" and "Intersect" methods for creating complex regions.
