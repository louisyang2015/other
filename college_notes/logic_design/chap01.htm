Number Systems and Conversions

Convert 5310 to binary:

53 = a0 + a1 * 2 + a2 * 22 + a3 * 23 + ...

so dividing both sides by 2, and the remainder is a0 , because a1 * 2, a2 * 22, and so on will divide by 2 evenly.

53 ÷ 2 = 26 remainder 1 (least significant digit because it corresponds to even or odd)
26 ÷ 2 = 13 remainder 0
13 ÷ 2 = 6 remainder 1
6 ÷ 2 = 3 remainder 0
3 ÷ 2 = 1 remainder 1
1 ÷ 2 = 0 remainder 1

answer: 11 0101

Convert 0.710 to binary

0.7 = a-1 * 2-1 + a-2 * 2-2 + a-3 * 2-3 + ...

so multiply by 2, and the units digit should be a-1 , because a-2 * 2-2 , a-3 * 2-3 , and so on will produce decimal results

0.7 * 2 = 1.4
0.4 * 2 = 0.8
0.8 * 2 = 1.6
0.6 * 2 = 1.2
0.2 * 2 = 0.4
0.4 * 2 = 0.8 (repeating pattern)

answer: 0.1 0110 0110 ...

Binary Codes for the 10 decimal digits

8-4-2-1 code means that 0111 is interpreted as 8*0 + 4*1 + 2*1 + 1*1

6-3-1-1 code means that 0111 is interpreted as 6*0 + 3*1 + 1*1 + 1*1

Excess-3 code is the 8421 code plus 3

The 2-out-of-5 code has exactly 2 ones out of the five bits

A gray code has the property that the codes for successive decimal digits differ in exactly one bit
