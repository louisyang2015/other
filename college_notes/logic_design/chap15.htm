Equivalent States

Two states are equivalent iff for every single input X, the outputs are the same and the next states are equivalent.

This definition is a circular one. To detect equivalence we go over a states table and any two states that have the same output are suspected of being equivalent. For every pair of these equivalent suspects, we look at their future states and eventually detect or reject equivalence.

Equivalent Sequential Networks

Two sequential networks are equivalent if they are capable of doing the same work. N1 is equivalent to N2 if for each state p in N1 there is a state q in N2 such that p ≡ q, and vice versa.

Say we are given two state tables. One table has four states named (A, B, C, D) while the other table has four states named (S0 , S1 , S2 , S3). We can discover whether the two networks are equivalent by considering all possible equivalence of states. We use a chart like we did before:

S0        
S1        
S2        
S3        
  A B C D

and if A and S0 are not equivalent, then we put an X in the (A, S0) box. If for every state in the (A, B, C, D) network we find an equivalent state in the (S0 , S1 , S2 , S3) network, then these two networks are equivalent.

 

Incompletely Specified State Tables

Sometimes, certain sequences never occur as inputs and we have don't care conditions in the states table for those input sequences. Incompletely specified tables can be filled so to simplify the network.

Say we are trying to implement the following network:

Equivalent State Assignments

Say we have three states S0 , S1 , and S2 . There are four ways (00, 01, 10, 11) to assign the first state, three ways to assign the second state, and two remaining ways to assign the third state, for a total of 24 ways to do state assignment.

The cost of the logic required to realize a sequential network is strongly dependent on the way this state assignment is made.

We do not have to check all 24 methods of assignment to find the cheapest implementation. If we interchange two columns in one of the given assignments, the cost of implementation should not change because interchanging columns is equivalent to re-labeling the flip-flops. So method 1 and method 3 in the above assignment table have the same cost.

If symmetrical flip-flops such as J-K flip-flops are used, complementing one or more columns of the state assignment will have no effect on the cost of realization. The reason is that J-K flip flops have two inputs that are opposite of one another anyways. For D flip-flips, complementing a column may result in adding an inverter at the flip-flop input. But even then we might avoid the inverter by say using NAND gates instead of AND gates. The conclusion is that complementing one or more columns of the state assignment will in general not change the cost of implementation much.

Two state assignments are equivalent if one can be derived from the other by permuting and complementing columns. Two state assignments which are not equivalent are said to be distinct.

There are only a few distinct assignments for 3-state and 4-state networks, so we can actually find the optimal assignment by trying them all.

But the number of distinct assignments increases rapidly with the number of states. By the time there are nine states, even a high speed computer will have problem trying all possible distinct assignments.

 

Guidelines for State Assignment

Assignment for two states are said to be adjacent if they differ in only one variable --- because they would then be adjacent on the K-map.

1. States which have the same next state for a given input should be given adjacent assignments

2. States which are the next states of the same state should be given adjacent assignments

3. States which have the same output for a given input should be given adjacent assignments.

First figure out all the adjacencies recommended. Then using a K-Map, satisfy as many of these adjacencies as possible.

Normally, the adjacencies from the first guideline should be given priority, followed by the adjacencies from the second guideline. The third guideline has the lowest priority. The following example will give more insight as to how these guidelines work.

So guideline 3 can gives the least adjacencies out of all three guidelines and therefore has the lowest priority. However guideline 3 can give 2 adjacencies if two states have totally identical output, and in such case guideline 3 should be given the same priority as guideline 2. If there are multiple outputs Z1 , Z2 , Z3 , ... then guideline 3 would have an even higher priority --- but then it's harder for two states to have the same output across the various Zs.

These guidelines can generate multiple results. So I think computer software that optimize networks work by first using the three guidelines to generate a number of state assignment possibilities, and then work out each possibility to find the best recommendation.
