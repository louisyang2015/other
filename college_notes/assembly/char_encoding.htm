The EOL marker for Dos

is two characters: the carriage return (CR, Ascii 13) followed by a line feed (LF, Ascii 10).

 

EOF

The EOF is not a Dos convention at all. Some OS prior to DOS does not count how many bytes there are in a text file, but relies on a EOF marker instead.

The "ctrl-Z" keyboard combo will generate an EOF marker. But it's mostly useless in modern day computing --- so removing the EOF marker from a text file is okay.

 

The EOF marker applies only for text files --- it doesn't apply to binary files at all.
