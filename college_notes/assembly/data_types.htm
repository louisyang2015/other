"Data Types"

The notion of "type" in assembly language is almost wholly a question of size.

To reserve bytes:

resb 64        ; reserve 64 bytes for the program stack

To declare a byte:

DB 6E

To give the allocation a name lable:

MyByte DB 07H        ; 8 bits in size

MyWord DW 0FFFFH        ; 16 bits in size

MyDouble DD 0B8000000        ; 32 bits in size

 

To use the variables:

Counter DB 0

MixTag DW 32

...

MOV BL, BYTE [Counter] ; move the zero into BL (move MEM[counter]) into BL
; with MASM, you don't need that "BYTE" because you already defined "Counter" to be a byte
; but with NASM, you would need that "BYTE" directive

MOV BL, BYTE MixTag
; MixTag is a word (2 bytes), BUT the use of "BYTE" moves just one byte of MixTag
; although this is peculiar, the assembler will not generate an error since there is no
; ambiguity

 

When operating on memory addresses, you sometimes need to tell the assembly are you operating on just a single byte at that memory address or a single word at that memory address.

NEG BYTE [BX]        ; Negates byte quantity at DS:BX

NEG WORD [BX]        ; Negates word quantity at DS:BX
