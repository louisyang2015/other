Arithmetic Operations

ADD DI, AX

Multiplication and Division --- IMUL, etc. --- these instruction have many forms depending on the size of the operands. The answer will might go into two different places because one register is sometimes not enough to hold the answer. (I think I don't have to worry about this stuff --- just let the C/C++ compiler generate the machine code).

 

INC AX

DEC BX

If AX = 0xFFFF, and you do "INC AX", the new AX is zero --- the carry flag is not set for this, but the zero flag will be set.

 

NEG BYTE [BX]        ; Negates byte quantity at DS:BX

NEG WORD [BX]        ; Negates word quantity at DS:BX

 

SHL <register/memory>, <count>  ; Shifts left "count" number of bits

SHR <register/memory>, <count> ; Shifts right "count" number of bits

On the 8086 and 8088, count can be 1 or the CL register; (starting with 286, count can be any immediate value from 1 to 255)

The last bit shifted out goes into the carry flag

 

Bitwise logical instructions

AND DX, SI

others: OR, XOR, NOT

Segment registers can not be used in any of the bitwise logical instructions.
